# Smart Shield とは

## ひとことで言うと

**サーバを狙う不審な相手を、ログから自動で見つけ出して、その場で締め出すソフトです。**

ウェブサイトやサーバを公開していると、知らない人 (実際は人ではなくほぼ自動の攻撃プログラム) が毎日のように

- 「もしかして admin というユーザでログインできるんじゃないか？」
- 「パスワードが書いてある `.env` ってファイルが残ってないかな？」
- 「管理画面 `/wp-admin/` が無防備に置いてあるかも？」

と片っ端から試してきます。当たれば乗っ取り、当たらなくてもサーバが重くなる原因になります。

Smart Shield は **その「試している様子」がログに残った瞬間を見つけ**、しつこく試している相手の住所 (IP アドレス) を **しばらくの間サーバへ入れなくする** ことで、こうした攻撃を未然に止めます。

## 何がうれしいか

| 困りごと | Smart Shield があると |
|---|---|
| 何度もパスワードを総当たりされる | しきい値を超えた相手を自動で締め出す |
| 知らないスキャナがサイトを舐めて回る | 過去のパターンに加え、AI が「いつもと違う動き」も見つけてくれる |
| 攻撃の対応に人手がかかる | 監視 → 判断 → ブロック → 解除 まで全部自動 |
| いつ・誰を・なぜブロックしたか後から追えない | すべてデータベースに残るので後から監査できる |
| 同じ相手が ban が解けるたびに何度も戻ってくる | 設定した回数を超えたら自動で **永久 ban** に格上げ |

## 何をしないか (大事)

- **正規のお客さんを意図的にブロックすることはありません。** 失敗を一定回数繰り返した相手だけが対象です。
- **個人情報の中身を読むことはありません。** 見ているのはサーバが出すアクセスログ (誰がいつ何にアクセスしたか) だけです。
- **未知の脆弱性を塞ぐソフトではありません。** あくまで「攻撃の試行」を遅らせる・止めるための仕組みです。OS やアプリのアップデートは別途必要です。

## 2 つのエンジン

Smart Shield には判定エンジンが 2 つ載っています。両方が独立に動き、片方が拾えなかった相手をもう片方が拾えるようになっています。

```
        ┌─────────────────────────────────────────────┐
ログ ─→  │  ① ルールエンジン (即時)                     │ ─→ ブロック
        │     - 「失敗を○分で△回」のような単純で速い判定  │
        └─────────────────────────────────────────────┘
                       │
                       ▼
        ┌─────────────────────────────────────────────┐
        │  ② AI エンジン (定期)                         │ ─→ ブロック
        │     - 数分ごとに直近のログをまとめて Claude へ │
        │     - 「ゆっくり時間をかけたスキャン」など     │
        │       単純ルールでは見抜けない攻撃も検出       │
        └─────────────────────────────────────────────┘
```

詳しくは [how-it-works.md](how-it-works.md) を見てください。

## イメージで掴む使い方

1. サーバに Smart Shield をインストールして、常時動かしておく。
2. 普段は何もしなくて OK。攻撃があれば自動で気付き、勝手にブロックして、時間が経てば勝手に解除する。
3. 何が起きたか知りたくなったら `smart-shield status` で「今ブロック中の相手と理由」を一覧で見られる。

## どこに置くソフトか

- ウェブサーバ (**Nginx / Apache** どちらも対応) の入っているマシン
- SSH で外からログインできるサーバ
- 社内の踏み台サーバ
- など、**インターネットからアクセスされる側のサーバ** に常駐させます。

クライアント PC (社員のノート PC など) には入れません。

### 標準で対応しているソフト

| 対象 | 拾うログ |
|---|---|
| OpenSSH | `/var/log/auth.log` など (Linux) / OpenSSH for Windows |
| Nginx | `access.log` (401/403、スキャナ探索 URL) |
| Apache | `access_log` + `error_log` (認証失敗、scanner、mod_security ブロック) |

これら以外のソフトも、フィルタを 1 つ書けば同じ枠組みで監視できます。

## 安心して試すための「お試しモード」

最初の設定で `dry_run: true` にしておくと、Smart Shield は **実際にはブロックしません**。
「もし本番モードだったらこの相手をこのタイミングでブロックしていた」という記録だけを残します。

しばらくこの状態で動かしてみて、

- ちゃんと悪い相手だけが捕まっているか
- 自社の正規アクセスを間違えてブロックしようとしていないか

を確認してから本番モードに切り替えるのが、現場で安全な進め方です。

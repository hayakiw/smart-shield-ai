# AI ログ解析について

Smart Shield は **Claude (Anthropic)** と **Gemini (Google)** の 2 つのプロバイダに対応しています。
コスト最優先なら Gemini 2.5 Flash、精度最優先なら Claude Sonnet がおすすめです。

## なぜ AI が必要なのか

ルールエンジン (正規表現で「失敗 N 回」のような判定) は速くて確実ですが、次のような攻撃を取りこぼします:

| ルールでは取りにくい例 | なぜ取りにくいか |
|---|---|
| **ゆっくり試す**: 10 分に 1 回ずつパスワードを試す | 時間窓内のカウントが閾値に届かない |
| **多数の IP に分散**: 1000 台のボットから 1 回ずつ試す | IP ごとには 1 回しか失敗しない |
| **新種ツールのスキャン**: 既知のシグネチャに一致しない URL を狙う | パターンを登録していない |
| **複合的なふるまい**: 単発では無害だが組み合わせて見ると攻撃の準備 | 行単位の一致では関連が見えない |

人間のセキュリティ担当者ならログを眺めて気付けるこれらの状況を、Smart Shield は **数分に 1 回 AI に読ませる** ことで自動化しています。

## 仕組み

```
   tail で集めたログ
     │
     ├─→ 直近 N 分の行をパス別バッファに溜める
     │
     │   (interval_seconds 経過)
     ▼
   ┌─────────────────────────────────────────┐
   │ AI プロバイダ (Claude or Gemini) へ送る   │
   │  - システム: SOC アナリストとして読む     │
   │  - ユーザ : 直近ログの抜粋               │
   └─────────────────────────────────────────┘
     │ JSON で {ip, reason, confidence} のリストが返る
     ▼
   confidence >= min_confidence  かつ
   whitelist に含まれない  かつ
   既にブロック中ではない
     │
     ▼
   ファイアウォールにブロックを登録
```

## プロバイダの選び方

設定ファイル (`config/shield.yaml`) で `ai.provider` を切り替えます:

```yaml
ai:
  provider: gemini              # anthropic | gemini
  model: gemini-2.5-flash       # 省略時は provider 別のデフォルト
```

| provider | デフォルトモデル | API キー環境変数 | 必要パッケージ |
|---|---|---|---|
| `anthropic` | `claude-sonnet-4-6` | `ANTHROPIC_API_KEY` | `anthropic` |
| `gemini` | `gemini-2.5-flash` | `GEMINI_API_KEY` (or `GOOGLE_API_KEY`) | `google-genai` |

両 SDK は **遅延 import** なので、使う方だけインストールすれば OK です:

```bash
pip install '.[gemini]'      # Gemini だけ
pip install '.[anthropic]'   # Claude だけ
pip install '.[all-ai]'      # 両方
```

## モデル別コスト感 (5 分間隔・60k 文字 / 回 で運用した場合)

正確な金額は各社価格表とトークン数で変動しますが、ざっくりの体感:

| モデル | 入力 100万トークン | 月額目安 | 特徴 |
|---|---|---|---|
| Claude Opus 4.7 | ~$15 | 万円台 | 精度最高、コスト重め |
| Claude Sonnet 4.6 | ~$3 | 数千円〜 | バランス型 |
| Claude Haiku 4.5 | ~$1 | 千円前後 | 軽量・高速 |
| **Gemini 2.5 Flash** | **~$0.30** | **数百円** | **コスト最優先で実用十分** |
| Gemini 2.5 Flash-Lite | ~$0.10 | 100 円前後 | 大量ログ環境向け |

**推奨**: 立ち上げ時は **Gemini 2.5 Flash** で十分。誤検知が気になる場合は Claude Sonnet または Gemini 2.5 Pro に上げる、という運用が現実的です。

## AI に渡しているプロンプト (要旨)

実物は [shield/ai_analyzer.py](../shield/ai_analyzer.py) にありますが、要点は:

- 「あなたは熟練の SOC アナリストです」
- 明確に悪意がある (またはほぼ確実に悪意がある) IP だけ挙げる
- プライベート IP は除外
- **確証がないものは出さない** (誤検知は重大なコスト)
- 出力は厳密な JSON のみ、前後に文章を付けない

実運用で最も大事なのは「誤検知させない」プロンプト圧であり、これにより `min_confidence` のしきい値 (デフォルト 0.75) と二重で誤判定を抑えています。

## プロバイダ別の最適化

| | Anthropic | Gemini |
|---|---|---|
| 構造化出力 | プロンプトで JSON 強制 + フェンス除去で安全側に | `response_mime_type="application/json"` を指定し API レベルで JSON 強制 |
| プロンプトキャッシュ | システムに `cache_control: ephemeral` を付与 (自動) | 短いシステムプロンプトは標準対応外。長い場合は Context Caching API を別途使用 |
| 温度 | デフォルト | `temperature=0.0` で固定 (再現性重視) |

## 料金とパフォーマンス

- 1 回の解析で送るログは `max_log_chars` (デフォルト 60,000 文字 ≒ 数百行) でクリップしています。
- インターバル `interval_seconds` (デフォルト 5 分) を伸ばすほどコストは下がりますが、攻撃検知の遅延は増えます。
- **Gemini 2.5 Flash** を使えば 1 日あたりおおむね **数十円** のオーダーで運用できます (ログ量・モデルにより変動)。

## 安全装置

AI が間違って正規ユーザをブロックしないように、以下の層が用意されています:

1. **whitelist**: 自社の IP / CIDR を `global.whitelist` に書いておけば、AI が指名しても無視されます。
2. **min_confidence**: AI 自身の自信度が高いものだけ採用 (低いものはログに残るだけ)。
3. **dry_run モード**: 最初のうちは実ブロックせず判定結果だけ記録。AI の癖を観察してから本番へ。
4. **ban_seconds**: AI 由来 ban の継続時間は別に設定可能。短めにしておけば、誤検知でも自動で解除されます。
5. **手動 unban**: いつでも `python -m shield unban <ip>` で即解除できます。

## AI を使わない選択肢

AI 解析は **オプション** です。`shield.yaml` で

```yaml
ai:
  enabled: false
```

にすればルールエンジンだけで動きます。AI を切ってもブロック機能そのものは普通に使えます。
コスト・外部依存・データ送信ポリシーを考慮して、必要に応じて切ってください。

## どんなログを Claude に送っているか

Smart Shield は **設定で指定したログファイルの中身** をそのまま送ります。
たとえば SSH ログや Nginx アクセスログには以下のような情報が含まれます:

- 接続元 IP アドレス
- 試したユーザ名
- アクセスした URL パス
- User-Agent
- 時刻

ログにアプリケーション固有の機微情報 (顧客のメールアドレス、個人を特定できる情報) が含まれている場合は、
Smart Shield に解析させるログを限定する、もしくは事前にマスク処理することを検討してください。
社内ポリシーで外部 API へのログ送信が禁止されている環境では `ai.enabled: false` のままが安全です。
